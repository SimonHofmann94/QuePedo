# Project Specification: Spanish Learning PWA (LingoApp)

## 1. Project Overview
We are building a mobile-first Progressive Web App (PWA) for learning Spanish. The app will help users build vocabulary, practice grammar with AI, and track their progress. It must be responsive (mobile and desktop) and installable as a PWA.

## 2. Technical Stack
- **Frontend:** Next.js (App Router), React, TypeScript.
- **Styling:** Tailwind CSS, Lucide React (icons), shadcn/ui (components).
- **Backend/DB:** Supabase (PostgreSQL, Auth, Row Level Security).
- **AI:** OpenAI API (gpt-4o-mini) called via Supabase Edge Functions.
- **State Management:** TanStack Query (React Query) for server state, Zustand for local app state.
- **Forms:** React Hook Form + Zod.

## 3. Database Schema (Supabase PostgreSQL)
*The app is multi-user. All tables must include `user_id` referencing `auth.users`.*

1.  **profiles**: (id, user_id, username, level [A1-C2], total_xp)
2.  **vocabulary**:
    - id (uuid)
    - term (string) - e.g., "La casa"
    - translation (string) - e.g., "The house"
    - context_sentence (text)
    - difficulty_rating (int) - Scale 1 (Easy) to 5 (Hard)
    - tags (array) - e.g., ["food", "travel"]
    - created_at
3.  **quiz_results**:
    - id
    - score (int)
    - quiz_type (string) - "vocab" | "grammar"
    - details (jsonb) - stores specific mistakes
4.  **grammar_progress**:
    - topic_id (string)
    - status (string) - "locked", "in_progress", "completed"

## 4. Functional Requirements

### Feature 1: Vocabulary Builder (Smart List)
- **Manual Entry:** Form to add Word, Translation, Example Sentence, and Difficulty Rating (1-5).
- **AI Entry:** Button "Generate Vocab by Topic". User types "Airport", AI generates 10 relevant words with definitions and context.
- **Search:** Real-time search bar filtering the user's list.

### Feature 2: Vocabulary Quiz
- **Settings:** Select Topic (or "All"), Direction (DE->ES or ES->DE), and Count (10, 20, 50).
- **Algorithm:** Prioritize words with higher `difficulty_rating` or older `last_practiced` dates.
- **UI:** Flashcard style or Multiple Choice.

### Feature 3: AI Grammar Exercises
- User selects a grammar concept (e.g., "Subjunctive") and a type:
    1.  **Cloze Test:** Text with missing words.
    2.  **Open Writing:** "Write a paragraph about your weekend." (AI corrects and explains errors).
    3.  **Listening:** Use Web Speech API or OpenAI TTS to read a sentence; user types what they hear.
    4.  **Speaking:** Use Web Speech API (STT); user reads a sentence, AI checks pronunciation match.

### Feature 4: Grammar Reference
- A structured library of static content (A1-C2).
- Accordion-style UI for easy mobile navigation.

### Feature 5: Global Translator
- **Floating Action Button (FAB):** Visible on all screens (bottom right).
- Opens a modal/drawer.
- Input text -> AI translates -> Button to "Save to Vocabulary".

### Feature 6: Culture Map
- Interactive Map (SVG or Library).
- Clicking a country (e.g., Mexico) shows a modal with "Culture Snippet" (generated by AI or static data).

### Feature 7: Dashboard
- **Stats:** Current Level (A1), Total Words Learned, Day Streak.
- **Charts:** Simple bar chart showing Quiz Scores over the last 7 days.

## 5. UI/UX Guidelines
- **Mobile First:** Large touch targets (buttons minimum 44px height).
- **Navigation:** Bottom Tab Bar for Mobile, Sidebar for Desktop.
- **Theme:** Clean, modern, vibrant colors (Spanish flag palette accents).
